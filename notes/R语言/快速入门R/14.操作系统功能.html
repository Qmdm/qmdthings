操作系统功能

R拥有相当广泛的设施来访问其运行的操作系统：这使得它可以用作脚本语言，而R本身也大 量使用这种能力，例如用于安装软件包

由于R自身的脚本需要在所有平台上运行，因此人们付出了相当大的努力来使脚本功能尽可能 平台无关。

文件与目录

R 提供了大量用于操控文件系统的函数，这些函数大多是 POSIX 标准工具的对应版本：

创建：使用 file.create 创建空文件，使用 dir.create 创建目录。
临时文件：tempfile 用于在 R 绘图或运算过程中产生临时文件或目录。
删除：file.remove 用于删除文件，unlink 则可以删除整个目录树。
列出内容：list.files (或 dir) 和 list.dirs 用于获取目录清单，支持正则表达式筛选；Sys.glob 则支持使用通配符进行选择。
获取信息：file.info 可返回文件的大小、是否为目录等详细元数据。
存在与权限检查：使用 file.exists、file.access 进行基本检查；file_test 为熟悉 Shell 脚本的用户提供了类似 POSIX test 命令的功能。
复制与链接：使用 file.copy 复制文件；使用 file.link 和 Sys.readlink 处理文件系统链接。
交互式选择：file.choose 允许用户手动选择文件。在 Windows 上，choose.files 和 choose.dir 提供了更灵活的图形界面选择功能。
显示与编辑：file.show 和 file.edit 会调用当前 R 环境（如 RGui 或 R.app）对应的设施来查看或编辑文件。

文件路径

R 很大程度上依赖操作系统来处理路径，但为了跨平台兼容性，该章特别指出了一些需要注意的差异：

大小写敏感性：POSIX 系统（如 Linux）区分大小写，而 Windows 和 macOS 默认不区分。
分隔符：Windows 支持使用正斜杠 / 或反斜杠 \，R 会自动处理其中的异常情况。
路径组装：file.path 是推荐的路径拼接方式，能确保生成符合当前系统的路径；basename 和 dirname 用于提取路径的文件名部分或目录部分。
规范化与扩展：pathexpand 用于执行波浪号（~）路径扩展；normalizePath 用于寻找文件的标准规范路径。
权限管理：使用 Sys.chmod 修改 POSIX 样式的权限。但注意，Windows 仅部分支持（如只读权限）。

系统命令

R 允许在代码中直接调用外部程序：

调用函数：system 和 system2 是主要工具。 system2 更加通用，更推荐用于编写跨平台代码。
跨平台差异：在 Windows 上，system 的行为与其他系统不同，Windows 用户可以使用专属的 shell 函数。
命令查找：Sys.which 可用于在系统路径中搜索特定的可执行命令。
引号处理：shQuote 可根据当前 OS 的要求自动为文件路径加上引号，以确保命令能正确执行。

压缩与归档

R 具有透明处理压缩文件的强大能力：

透明读取：R 通过 “连接” (Connections) 机制读取文件，file 函数能根据文件的“幻数” (magic header) 自动识别压缩格式。
支持格式：包括长期支持的 gzip，以及压缩率更高但速度较慢的 bzip2 和 xz。 R 也可以处理 lzma 压缩。
归档管理：对于打包文件（如 tar 或 zip），可以使用 untar 和 unzip 进行列表查看和解压。 创建 zip 文件通常需要借助外部程序。

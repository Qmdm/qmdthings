#数组与矩阵

## 数组

数组可以看作是多个带下标的数据条目的集合，例如数值型数据。R 语言提供了创建和处理数组的简便方法，尤其擅长处理矩阵这种特殊情况。

维度向量(数组索引向量)是指由非负整数组成的向量。如果它的长度为 k，则该数组为 k 维数组，例如，矩阵是二维数组。维度索引从 1 开始，一直到维度向量中给出的值。

只有当向量的 dim 属性也为向量时，R 才能将其用作数组。例如，假设 z 是一个包含 1500 个元素的向量。

```bash
> dim(z)<-c(3,5,100)
```

赋予它 dim 属性，使其可以被视为 3×5×100 的数组。

数据向量中的值按照 FORTRAN 中出现的顺序（即“列优先顺序”）在数组中给出，也就是最先移动第一个下标，最后移动最后一个下标。

例如，如果数组 a 的维度向量为 `c(3,4,2)`，则 a 中有 3 × 4 × 2 = 24 个条目，数据向量按 `a[1,1,1]`、`a[2,1,1]`、...、`a[2,4,2]`、`a[3,4,2]` 的顺序保存它们。

数组可以是一维的：这类数组通常与向量的处理方式相同（包括打印时），但例外情况可能会造成混淆。

## 数组索引

**可以通过给出数组名称，后跟用逗号分隔的方括号中的下标来引用数组中的各个元素。**

更一般地，可以通过给出一系列索引向量来指定数组的子部分。代替下标；但是，如果任何索引位置被赋予一个空索引向量，则取该下标的整个范围。

注意是`a[1,1,1]`,而不是`a[1][1][1]`

按此顺序。`a[,,]` 代表整个数组，这与完全省略下标并单独使用 a 的效果相同。

对于任何数组，例如 Z，维度向量可以显式地引用为 dim(Z)（在赋值语句的两侧）。

此外，如果数组名仅包含一个下标或索引向量，则仅使用数据向量的对应值；在这种情况下，维度向量将被忽略。

```bash
>a<-c(1,2,3,4,5,6,7,8)
>dim(a)<-c(2,4)
>a
     [,1] [,2] [,3] [,4]
[1,]    1    3    5    7
[2,]    2    4    6    8
>a[5]
5
>a[1,]
[1] 1 3 5 7
```

## 索引矩阵

除了可以在任何下标位置使用索引向量外，还可以使用单个索引矩阵来将数量向量分配给数组中不规则的元素集合，或者将不规则集合提取为向量。

```bash
> z<-c(1:9)
> dim(z)<-c(3,3)
> z
     [,1] [,2] [,3]
[1,]    1    4    7
[2,]    2    5    8
[3,]    3    6    9
> index<-c(
+ 1,2,
+ 3,1,
+ 2,1
+ )
> dim(index)<-c(3,2)
> index
     [,1] [,2]
[1,]    1    1
[2,]    2    2
[3,]    3    1
>z[index]
[1] 1 5 3
```

索引矩阵中不允许出现**负索引**。允许出现 NA值和零值：索引矩阵中包含零的行将被忽略，包含 NA 值的行将在结果中产生 NA 值。

```bash
> x<-matrix(0,2,2)
> x
     [,1] [,2]
[1,]    0    0
[2,]    0    0
> index<-cbind(1:2,2:1) #Column Bind按列合并
> index
     [,1] [,2]
[1,]    1    2
[2,]    2    1
> x[index]<-1
> x
     [,1] [,2]
[1,]    0    1
[2,]    1    0
> indexb<-cbind(c(1,1),c(2,2))
> indexb
     [,1] [,2]
[1,]    1    2
[2,]    1    2
> N<-crossprod(index,indexb) # Cross Product 矩阵叉乘 A^TB
> N
     [,1] [,2]
[1,]    3    6
[2,]    3    6
> K<- table(c(1,2,3),c(4,5,6)) # 以两个向量的index提取两个数作为结果的行数与列数计算频数
> K

    4 5 6
  1 1 0 0
  2 0 1 0
  3 0 0 1
>
```

## array()函数

除了给向量结构添加 dim 属性之外，还可以使用`array()`函数从向量构造数组，其形式为：

```bash
> Z<-array(data_vector,dim_vector)
> Z<-array(h,dim=c(3,4,2))
> Z<-h;dim(Z)<-c(3,4,2)
```

然而，如果 h 小于 24，则其值会从头开始重新循环，但 `dim(h) <- c(3,4,2)` 会提示长度不匹配的错误。举个极端但常见的例子

```bash
> z=c(1:5)
> dim(z)=c(2,3)
Error in dim(z) = c(2, 3) :
  dims [product 6] do not match the length of object [5]

> z=array(c(1:5),c(2,3))
> z
     [,1] [,2] [,3]
[1,]    1    3    5
[2,]    2    4    1
>
```

此时 `dim(Z)` 表示维度向量 `c(3,4,2)`，`Z[1:24]` 表示 h 中的数据向量，而下标为空的 Z[] 或没有下标的 Z 表示整个数组。

数组可用于算术表达式，其结果是一个由对数据向量逐元素运算形成的数组。**操作数的维度属性通常需要相同**，这构成了结果的维度向量.

## 混合向量与数组运算

影响向量和数组混合计算的逐元素运算的具体规则比较特殊，很难在参考资料中找到。根据我们的经验，以下方法较为可靠。

- 表达式计算从左向右
- 如果两个向量长度不一，R 不会报错，而是把短的那个“头尾相接”重复使用，直到填满长的那个。
- 如果你让两个数组（或矩阵）做加减乘除，它们的“形状”必须一模一样。
- 向量可以和数组运算，但向量不能比数组的元素总数还多。
- 如果存在数组结构，并且没有发生错误或强制转换为向量，则结果是一个数组结构，其数组操作数具有共同的 dim 属性。

也就是说**向量可以被自动循环填充,改变形状,但是数组不行.**

## 数组的广义转置

对于二维数组,你可以使用`t()`进行转置

```bash
>a<-matrix(1:9,c(3,3))
>a
     [,1] [,2] [,3]
[1,]    1    4    7
[2,]    2    5    8
[3,]    3    6    9
>t(a)
     [,1] [,2] [,3]
[1,]    1    2    3
[2,]    4    5    6
[3,]    7    8    9
```

而对于高维数组则需使用`aperm()`

```bash
> a<-array(1:8,dim=c(2,2,2))
> a
, , 1

     [,1] [,2]
[1,]    1    3
[2,]    2    4

, , 2

     [,1] [,2]
[1,]    5    7
[2,]    6    8
> aperm(a,prem=c(3,2,1)) # Array Permutation 重新排列多维数组的维度顺序。
, , 1

     [,1] [,2]
[1,]    1    3
[2,]    5    7

, , 2

     [,1] [,2]
[1,]    2    4
[2,]    6    8

```

函数 aperm(a, perm) 可用于置换数组 a。参数 perm 必须是整数 {1, ..., k} 的一个置换，其中 k 是数组 a 中下标的个数。函数的结果是一个与 a 大小相同的数组，但原数组的第 j 个维度变为新的第 j 个维度。最简单的理解方式是将此操作视为矩阵转置的推广.

## 矩阵工具

### 矩阵乘法

如上所述，`t(X)` 是矩阵转置函数。函数 `nrow(A)` 和 `ncol(A)` 分别返回矩阵 A 的行数和列数。

运算符 `%*%` 用于矩阵乘法。`n×1` 或 `1×n` 矩阵当然也可以用作 `n` 维向量，如果上下文合适的话。反之，矩阵乘法表达式中出现的向量会自动提升为行向量或列向量，具体取决于乘法上的一致性

```bash
> A * B # 逐元素乘法
> A %*% B # 矩阵乘法
> x %*% A %*% x # 二次型
```

函数 crossprod() 用于计算“交叉积”，也就是说，crossprod(X, y) 与 t(X) %\*% y 等效，但前者效率更高。如果省略 crossprod() 的第二个参数，则其值与第一个参数相同。

diag() 函数的含义取决于其参数。diag(v)，其中 v 是一个向量，返回一个对角矩阵，其对角线元素为该向量的元素。另一方面，diag(M)，其中 M 是一个矩阵，返回矩阵 M 的主对角线元素向量。这与 Matlab 中 diag() 函数的约定相同。此外，需要注意的是，如果 k 是一个数值，则 diag(k) 表示 k×k 的单位矩阵！

## 线性方程组与反演

解线性方程组是矩阵乘法的逆运算。

```bash
>b<-A %*% x
```

一个线性方程组$Ax=b$

只给出了 A 和 b，向量 x 是该线性方程组的解。在 R 中，

```bash
>b <-A %*% x
>slove(A,b)
```

或者说求解A的逆矩阵

```bash
>solve(A)
```

但很少需要这样做。从数值上看，计算 `x <- solve(A) %*% b` 而不是 `solve(A,b)` 既效率低下，也可能不稳定。

在多元计算中使用的二次型 `xAx`` 应该通过类似 `x %\*% solve(A,x)` 的方式计算，而不是计算 A 的逆。

注意，`x %*% x` 的含义不明确，它可以表示 `xx` 或 `xx`，其中 `x` 是列向量。在这种情况下，较小的矩阵似乎被隐式地解释为最终结果，因此标量 `xx` 在此情况下即为结果。矩阵 `xx` 可以通过 `cbind(x) %*% x` 或 `x %*% rbind(x)` 计算，因为 `rbind()` 或 `cbind()` 的结果始终是一个矩阵。然而，计算 `xx` 或 `xx` 的最佳方法分别是使用 `crossprod(x)` 或 `x %o% x`。

更好的方法是构造一个矩阵平方根 B，其中 A = BB，并找到 By = x的解的平方长度，或许可以使用 A 的 Cholesky 分解或特征值分解。

## 奇异值分解与行列式

函数 `svd(M)` 接受一个任意矩阵参数 `M`，并计算 `M`的奇异值分解。该分解包含三个矩阵：一个列空间与 `M` 相同的正交列矩阵`U`，一个列空间与 `M` 相同的正交列矩阵`V`，以及一个对角元素均为正数的矩阵 `D`，其中 `M` 满足 `M = U %*% D %*% t(V)`。`D` 实际上以对角元素向量的形式返回。`svd(M)` 的结果实际上是一个包含三个元素的列表，分别命名为 `d`、`u` 和 `v`，其含义显而易见。

R 语言内置函数 det 可以计算行列式（包括符号），还可以计算另一个行列式（给出符号和模，也可以选择对数刻度）。

```bash
> a
     [,1] [,2] [,3]
[1,]    1    4    7
[2,]    2    5    8
[3,]    3    6    9
> svd(a)
$d
[1] 1.684810e+01 1.068370e+00 5.039188e-17

$u
           [,1]        [,2]       [,3]
[1,] -0.4796712  0.77669099  0.4082483
[2,] -0.5723678  0.07568647 -0.8164966
[3,] -0.6650644 -0.62531805  0.4082483

$v
           [,1]       [,2]       [,3]
[1,] -0.2148372 -0.8872307 -0.4082483
[2,] -0.5205874 -0.2496440  0.8164966
[3,] -0.8263375  0.3879428 -0.4082483
```

```bash
> absdetM <- prod(svd(M)$d) # Singular Value Decomposition
> absdet <- function(M) prod(svd(M)$d)
```

## 最小二乘拟合与QR分解

函数 lsfit() 返回一个列表，其中包含最小二乘拟合过程的结果。例如，以下赋值语句：

```bash
>ans<-lsfit(x,y)
```

该函数给出最小二乘拟合的结果，其中 y 是观测值向量，X 是设计矩阵。更多详情请参阅帮助文档，以及后续函数 ls.diag()，该函数可用于回归诊断等操作。请注意，总均值项会自动包含在内，无需显式地将其作为 X 的列添加。此外，对于回归建模，您几乎总是会优先选择使用 lm(.) 函数，而不是 lsfit() 函数。

另一个密切相关的函数是 qr() 及其相关函数。请考虑以下赋值语句。

```bash
> xplus <-qr(x)
> b <- qr.coef(Xplus, y)
> fit <- qr.fitted(Xplus, y)
> res <- qr.resid(Xplus, y)
```

这些函数分别计算 y 在 fit 中的正交投影到 X 的范围内、在 res 中的正交补上的投影以及投影的系数向量，其中 b 本质上是 Matlab 的“反斜杠”运算符的结果。

并不假定 X 具有满列秩。冗余数据将在发现后立即被移除。

这种方法是执行最小二乘计算的较旧、底层方法。虽然在某些情况下仍然有用，但现在通常会被统计模型功能所取代.

## 形成分块矩阵 cbind() 和 rbind()

cbind() 通过水平（或按列）方式将矩阵绑定在一起来形成矩阵，而 rbind() 通过垂直（或按行）方式将矩阵绑定在一起来形成矩阵。

rbind() 或 cbind() 的结果始终是矩阵形式。因此，cbind(x) 和 rbind(x) 可能是将向量 x 分别显式地视为列矩阵或行矩阵的最简单方法。

## 数组的连接函数 c()

需要注意的是，cbind() 和 rbind() 是会保留 dim 属性的连接函数，而基本的 c() 函数则不会，它会清除数值对象的所有 dim 和 dimnames 属性。这本身有时也很有用。

将数组强制转换回简单向量对象的官方方法是使用 as.vector()。

```bash
vec <- as.vector(x)
```

然而，使用只带一个参数的 c() 函数也能达到类似的效果，其副作用就在于此：

```bash
>vec<- c(x)
```

## 各因子的频率表

回想一下，一个因子定义了一个分组。类似地，一对因子定义了一个双向交叉分类，依此类推。`table()` 函数允许根据等长因子计算频数表。如果有 k 个因子参数，则结果是一个 k 维频数数组。
例如，假设 statef 是一个因子，它给出数据向量中每个条目的状态码。

```bash
> statefr <- table(statef)
```

以 statefr 形式给出样本中每个状态的频率表。频率按因子的 levels 属性排序和标记。这个简单的例子等价于，但比以下情况更方便：

```bash
> statefr <- tapply(statef, statef, length)
# Tag + Apply 将每一组用标签分组然后对每一组应用一个函数
```

进一步假设 incomef 是一个因子，它为数据向量中的每个条目给出一个适当定义的“收入类别”，例如使用 cut() 函数：

```bash
> factor(cut(incomes, breaks = 35+10*(0:7))) -> incomef # Cut函数是离散变量与连续变量的桥梁
```


使用交互式环境R

你可以轻而易举的更改交互式R环境的提示符 详见 更改交互式环境的提示符

$ mkdir work
$ cd work
$ R
> q()

此时系统会询问您是否要保存 R 会话中的数据。在某些系统中，这会弹出一个对话框；而在其他系统中，您会收到一条文本提示，您可以回复“是”、“否”或“取消”（只需输入一个字母即可），以决定是在退出前保存数据、不保存而退出，还是返回 R 会话。已保存的数据将在以后的 R 会话中可用。

>la

帮助与用例

R 语言内置了类似于 UNIX 系统 man 命令的帮助功能。要获取有关特定函数（例如 solve）的更多信息，可以使用以下命令：

> help(solve)
> # 或者是这样
>  ?solve

对于使用特殊字符指定的功能，参数必须用双引号或单引号括起来，使其成为“字符串”：对于一些具有语法意义的单词（例如 if、for 和 function）来说，这也是必要的。两种引号形式都可以用来转义另一种，例如字符串“It’s important”。我们的惯例是优先使用双引号。

> help("[[")

在大多数 R 安装中，运行以下命令即可获得 HTML 格式的帮助文档：

# 你可能需要这个命令来指定你的浏览器
> options(browser="librewolf")
> help.start()

这将启动一个 Web 浏览器，允许用户通过超链接浏览帮助页面。在 UNIX 系统上，后续的帮助请求将发送到基于 HTML 的帮助系统。help.start() 加载的页面中的“搜索引擎和关键字”链接尤其有用，因为它包含一个高级概念列表，可以搜索所有可用函数。这可以帮助您快速了解 R 语言的概况，并掌握其丰富的功能。

help.search 命令（或 ??）允许以多种方式搜索帮助。例如，

> ??solve

请尝试使用 ?help.search 查看详细信息和更多示例。帮助主题中的示例通常可以通过以下命令运行。

> example(topic = solve)

Windows 版 R 还有其他可选的帮助系统：使用

> ?help

R命令与大小写敏感

从技术上讲，R 是一种语法非常简单的表达式语言。它区分大小写*,R 名称中可以使用的符号集取决于操作系统和运行 R 的国家/地区（严格来说，取决于所使用的区域设置）。通常允许使用所有字母数字符号（在某些国家/地区，这包括带重音符号的字母），以及句点“.”和下划线“_”。但名称必须以句点“.”或字母开头，如果以句点“.”开头，则第二个字符不能是数字。名称的长度实际上没有限制。**

基本命令由表达式或赋值组成。如果命令中包含表达式，则会对其进行求值并打印出来（除非明确设置为不可见），但结果会丢失。赋值也会对表达式进行求值，并将结果赋值给一个变量，但结果不会自动打印出来。

对于可移植的 R 代码（包括 R 包中使用的代码），应该只使用 A–Z、a–z 和 0–9。

命令之间可以用分号（‘;’）或换行符分隔。 基本命令可以用花括号（‘{’ 和 ‘}’）组合成一个复合表达式。

注释几乎可以放在任何位置，以井号（‘#’）开头，行尾的所有内容都是注释。 类似交互式语言的python

如果一行末尾的命令不完整，R 默认会给出不同的提示符。

+

在第二行及后续行继续读取输入，直到命令语法正确为止。用户可以更改此提示符。我们通常会省略继续提示符，并通过简单的缩进来指示后续操作。

回忆与纠正

在许多 UNIX 版本和 Windows 系统中，R 提供了一种机制来调用和重新执行之前的命令。可以使用键盘上的垂直方向键在命令历史记录中前后滚动。找到某个命令后，可以使用水平方向键在命令内移动光标，并使用 DEL 键删除字符或使用其他键添加字符。更多详情请参见附录 C [命令行编辑器]，第 90 页。

UNIX 系统下的调用和编辑功能具有高度可定制性。您可以通过阅读 readline 库的手册条目来了解如何进行这些操作。

此外，Emacs 文本编辑器还提供了更通用的支持机制（通过 ESS，Emacs Speaks Statistics），用于与 R 进行交互式操作。有关更多信息，请参阅 R FAQ 中的“R 和 Emacs”部分。

从文件执行命令source() 或 将输出重定向到文件 sink()

如果命令存储在外部文件中，例如工作目录下的 commands.R 文件，则可以在 R 会话中随时使用命令执行这些命令。

> source("commands.R")

对于 Windows 系统，Source 也位于“文件”菜单中。函数接收器，

> sink("record.lis")

将会把控制台的所有后续输出重定向到外部文件 record.lis。该命令

> sink()

再次将其恢复到主机上。

数据永久性与 对象显示objects()/ls() 移除rm()

R 语言创建和操作的实体被称为对象。这些对象可以是变量、数字数组、字符串、函数，或者由这些组件构建的更一般的结构。

在 R 会话期间，对象会按名称创建和存储（我们将在下一节讨论此过程）。R 命令

> objects()

（或者，可以使用 ls() 函数）显示当前存储在 R 中的（大多数）对象的名称。当前存储的对象集合称为工作区。

可以使用 rm 函数删除对象：

> rm(x, y, z, ink, junk, temp, foo, bar)

在 R 会话期间创建的所有对象都可以永久存储在一个文件中，以便在以后的 R 会话中使用。每次 R 会话结束时，您都可以选择保存所有当前可用的对象。

如果您选择保存，这些对象将被写入当前目录下的名为 .RData 的文件中，而会话中使用的命令行将被保存到名为 .Rhistory 的文件中。

当稍后从同一目录启动 R 时，它会从此文件重新加载工作区。同时，相关的命令历史记录也会重新加载。

建议您为使用 R 进行的分析使用单独的工作目录。在分析过程中，创建名为 x 和 y 的对象是很常见的。这类名称在单个分析的上下文中通常有意义，但如果多个分析在同一目录中进行，则很难确定它们的含义。

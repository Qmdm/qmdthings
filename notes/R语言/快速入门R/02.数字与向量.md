# 数字与向量

R 语言使用命名数据结构。最简单的数据结构是**数值向量:一个包含有序数字集合的单一实体。**

例如，要创建一个名为 x 的向量，其中包含五个数字：10.4、5.6、3.1、6.4 和 21.7，可以使用 R 命令

```bash
> x <- c(10.4, 5.6, 3.1, 6.4, 21.7)
```

这是一个使用函数 c() 的赋值语句，在本上下文中，该函数可以接受任意数量的向量参数，其值是通过将其参数首尾连接而得到的向量。

> [!TIP]
> 在 R 语言中，c() 是 Combine（结合）或者 Concatenate（连接）的缩写。

表达式中单独出现的数字被视为长度为 1 的向量。请注意，赋值运算符（'<-'）由两个紧挨着的字符 '<'（“小于”）和 '-'（“减”）组成，它指向接收表达式值的对象。

**在大多数情况下，可以使用运算符 '=' 作为替代。**
也可以使用 `assign()` 函数进行赋值。与上述赋值方式等效的方法是：

```bash
>assign("x",c(10.4, 5.6, 3.1, 6.4, 21.7))
```

常用的运算符 <- 可以看作是实现此目的的语法快捷方式。

**赋值也可以反向进行，只需对赋值运算符进行简单的更改即可。**

因此，同样的赋值也可以使用以下方式实现：

```bash
> c(10.4, 5.6, 3.1, 6.4, 21.7) -> x
```

如果将表达式用作完整命令，则其值会被打印出来并丢失。所以现在如果我们使用命令

```bash
> 1/x
```

终端会打印出这五个值的倒数（当然，x 的值保持不变）。

```bash
> y <- c(x, 0, x)
```

会创建一个包含 11 个元素的向量 y，其中 x 是两个副本，中间位置为零。

## 向量运算

向量可用于算术表达式，此时运算将逐个元素进行。同一表达式中出现的向量不必长度相同。如果长度不同，则表达式的值是一个与表达式中最长向量长度相同的向量。

**表达式中较短的向量会根据需要重复使用（可能使用小数），直到它们的长度与最长向量的长度相同。**

特别是，常量会被重复使用。因此，对于上述赋值，命令

```bash
> x
[1] 10.4  5.6  3.1  6.4 21.7

>y
 [1] 10.4  5.6  3.1  6.4 21.7  0.0 10.4  5.6  3.1  6.4 21.7

> v <- 2*x + y + 1

Warning message:
In 2 * x + y :
  longer object length is not a multiple of shorter object length
> v
 [1] 32.2 17.8 10.3 20.2 66.1 21.8 22.6 12.8 16.9 50.8 43.5
```

- 循环对齐,也就是说

```r
x = c(
10.4,5.6,3.1,6.4,21.7,10.4,
10.4,5.6,3.1,6.4,21.7,10.4,
10.4
)
```

基本算术运算符包括常用的 +、-、\*、/ 和 ^（用于求幂）。

此外，所有常用的算术函数也都可用，例如 log、exp、sin、cos、tan、sqrt 等，它们都具有通常的含义。

max 和 min 分别用于选择向量中的最大值和最小值。

range 是一个函数，其值是一个长度为 2 的向量，即 c(min(x), max(x))。length(x) 表示 x 中元素的个数，

sum(x) 表示 x 中元素的总和，prod(x) 表示它们的乘积。

两个统计函数分别是 mean(x) 和 var(x)，它们计算样本均值，即 sum(x)/length(x)；以及 var(x)，它们给出sum((x-mean(x))^2)/(length(x)-1)或者样本方差。

**如果 var() 的参数是一个 n×p 矩阵，则其值为一个 p×p 样本协方差矩阵，该矩阵的行被视为独立的 p 元样本向量。**

sort(x) 返回一个与 x 大小相同的向量，其中的元素按升序排列；但是还有其他更灵活的排序功能可用（请参阅 order() 或 sort.list()，它们会生成一个排列来进行排序）。

请注意，即使给定多个向量，`max` 和 `min` 函数也会选择其参数中的最大值和最小值。并行的最大值函数 `pmax` 和最小值函数 `pmin` 返回一个向量（长度等于其最长参数的长度），该向量的每个元素包含输入向量中该位置的最大（最小）元素。

大多数情况下，用户无需关心数值向量中的“数字”是整数、实数还是复数。内部计算以双精度实数进行，如果输入数据是复数，则以双精度复数进行。

**要处理复数，需要显式地指定复数部分。**

因此`sqrt(-17)``会返回 `NaN` 值并发出警告，但是`sqrt(-17+0i)`将以复数形式进行计算。

## 生成正则序列 :/seq/rep

R 语言提供了许多生成常用数字序列的功能。

例如，1:30 是一个向量 c(1, 2, ..., 29, 30)。
冒号运算符在表达式中具有很高的优先级，因此，例如 2\*1:15 是一个向量 c(2, 4, ..., 28, 30)。令 n <- 10，并比较序列 1:n-1 和 1:(n-1)。

构造 30:1 可用于反向生成序列。

**seq() 函数是一个更通用的序列生成工具。**
它有五个参数，每次调用只能指定其中的一部分。如果指定了前两个参数，则它们分别指定序列的起始位置和结束位置；如果只指定了这两个参数，则结果与冒号运算符相同。也就是说，seq(2,10) 和 2:10 是同一个向量。

seq() 函数以及许多其他 R 函数的参数也可以用命名形式给出，在这种情况下，参数出现的顺序无关紧要。前两个参数可以命名为 from=value 和 to=value；因此，seq(1,30)、seq(from=1, to=30) 和 seq(to=30, from=1) 都等同于 1:30。seq() 函数的后两个参数可以命名为 by=value 和 length=value，分别指定序列的步长和长度。如果这两个参数都没有指定，则默认使用 by=1。

**第五个参数可以命名为 along=vector，通常用作唯一参数来创建序列 1, 2, ..., length(vector)，或者如果向量为空（向量可能为空），则创建空序列。**

一个相关的函数是 **rep()(replicates复制)，它可以用于以各种复杂的方式复制对象。**

最简单的形式是：

```bash
# x = c(1,2)
> s5 <- rep(x, times=5)
# s5 = (1,2,1,2,1,2,1,2,1,2)
```

**这将把五个 x 副本首尾相连地放置在 s5 中。** 另一个有用的版本是

```bash
# x = c(1,2)
> s6 <- rep(x, each=5)
# s5 =c(1,1,1,1,1,5,5,5,5,5)
```

**它将 x 的每个元素重复五次，然后再进行下一个元素。**

## 逻辑向量

除了数值向量，R 语言还允许操作逻辑量。逻辑向量的元素可以取值为 TRUE、FALSE 和 NA（NA 表示“不可用”，详见下文）。前两者通常分别简写为 T 和 F。但请注意，**T 和 F 只是变量，默认值分别为 TRUE 和 FALSE**，但它们并非保留字，因此用户可以修改其值。所以，您应该始终使用 TRUE 和 FALSE。

逻辑向量由条件生成。例如

```bash
> temp <- x > 13
```

将 temp 设置为与 x 长度相同的向量，其中 FALSE 对应于 x 中条件不满足的元素，TRUE 对应于条件满足的元素。

逻辑运算符包括 <、<=、>、>=、== 表示完全相等，以及 != 表示不等。此外，如果 c1 和 c2 是逻辑表达式，则 c1 & c2 表示它们的交集（“与”），c1 | c2 表示它们的并集（“或”），而 !c1 表示 c1 的否定。

逻辑向量可以用于普通算术运算，在这种情况下，它们会被强制转换为数值向量，FALSE 变为 0，TRUE 变为 1。然而，在某些情况下，**逻辑向量与其强制转换为数值的对应物并不等价**

在某些情况下，向量的各个分量可能并不完全已知。当某个元素或值“不可用”或统计学意义上的“缺失值”时，可以通过赋予其特殊值 NA 来为其预留向量中的一个位置。通常，对 NA 进行的任何运算结果仍然是 NA。这条规则的理由很简单：如果运算的描述不完整，则无法得知结果，因此结果也不可用。

函数 is.na(x) 返回一个与 x 大小相同的逻辑向量，当且仅当 x 中对应的元素为 NA 时，该逻辑向量的值为 TRUE。

```bash
> z <- c(1:3,NA); ind <- is.na(z)
```

注意，逻辑表达式 x == NA 与 is.na(x) 截然不同，因为 NA 实际上并非一个值，而是一个表示数量不可用的标记。因此，**x == NA 是一个与 x 长度相同的向量，其所有元素均为 NA，因为逻辑表达式本身是不完整的，因此是不可判定的。**

请注意，还有第二种“缺失值”，即数值计算产生的所谓 **“非数字”（NaN)** 值。 例如：

```bash
> 0/0
> NAN
```

```bash
> is.na(0/0);is.nan(0/0)
[1] TRUE
[1] TRUE
```

总之，is.na(xx) 对 NA 值和 NaN 值都返回 TRUE。为了区分这两者，is.nan(xx) 仅对 NaN 值返回 TRUE。

缺失值有时会打印为当字符向量打印时没有引号。

## 字符向量

字符数量和字符向量在 R 语言中经常使用，例如用作图表标签。需要时，它们用双引号分隔的字符序列表示，例如，“x 值”、“新迭代结果”。

字符串输入时可以使用匹配的双引号 (") 或单引号 (')，但打印时使用双引号（有时也可能不使用引号）。它们使用 C 风格的转义序列，以反斜杠 (\) 作为转义字符，因此反斜杠 (\) 输入和打印为 \\，双引号内的单引号 (") 输入为 \"。其他常用的转义序列包括换行符 (\n)、制表符 (\t) 和退格符 (\b)——**完整列表请参见“?Quotes”。**

可以使用 c() 函数将字符向量连接成一个向量；它们的使用示例将会频繁出现。

**paste() 函数接受任意数量的参数，并将它们逐一连接成字符串。** 参数中的任何数字都会被强制转换为字符串，转换方式与打印时相同。默认情况下，结果中的参数之间用一个空格分隔，但可以通过指定参数 sep=string 来更改分隔符，该参数会将分隔符更改为字符串，字符串可能为空。

例如

```bash
> labs <- paste(c("X","Y"), 1:10, sep="") 将 labs 转换为字符向量
```

```bash
c（“X1”，“Y2”，“X3”，“Y4”，“X5”，“Y6”，“X7”，“Y8”，“X9”，“Y10”）
```

特别要注意的是，这里也存在短列表的循环利用；因此 c("X", "Y") 重复 5 次以匹配序列 1:10。

## 索引向量

选择和修改数据集的子集

可以通过在向量名称后附加一个用方括号括起来的索引向量来选择向量的元素子集。更一般地，**任何求值为向量的表达式，都可以通过在表达式后紧跟一个用方括号括起来的索引向量来选择其元素子集。**

此类索引向量可以是以下四种不同类型中的任何一种。

1. **逻辑向量。** 在这种情况下，索引向量会被循环使用，直到长度与待选元素的向量相同。索引向量中值为 TRUE 的元素会被选中，值为 FALSE 的元素则会被忽略。例如

```bash
> y <- x[!is.na(x)]
```

创建（或重新创建）一个对象 y，其中包含 x 的非缺失值，顺序与 x 相同。请注意，如果 x 存在缺失值，则 y 的长度将小于 x。

```bash
> (x+1)[(!is.na(x)) & x>0] -> z
```

创建一个对象 z，并将向量 x+1 的值放入其中，这些值在 x 中对应的值既非缺失又为正。

2. **一个包含正整数的向量。** 在这种情况下，索引向量中的值必须位于集合 {1, 2, ..., length(x)} 中。向量中对应的元素按顺序被选中并连接起来，得到结果。索引向量的长度可以是任意的，结果的长度与索引向量的长度相同。例如，x[6] 是 x 的第六个分量。

```bash
> x[1:10]
```

选择 x 的前 10 个元素（假设 length(x) 不小于 10）。

```bash
> c("x","y")[rep(c(1,2,2,1), times=4)]
```

（虽然不太可能发生）生成一个长度为 16 的字符向量，其中包含重复四次的“x”、“y”、“y”、“x”。

3. **一个负整数向量。** **这样的索引向量指定要排除而不是包含的值。** 因此

```bash
> 和 <- x[-(1:5)]
```

4. **一个字符串向量。** 这种情况仅适用于具有 `names` 属性来标识其组成部分的对象。在这种情况下，`names` 向量的子向量可以像上面第 2 项中的正整数标签一样使用。

```bash
> fruit <- c(5, 10, 1, 20)
>  names(fruit) <- c("orange", "banana", "apple", "peach")
> lunch <- fruit[c​​("apple","orange")]
```

这样做的好处是，字母数字名称通常比数字索引更容易记忆。这一点在处理数据框时尤其有用，我们稍后会看到。

索引表达式也可以出现在赋值语句的接收端，在这种情况下，赋值操作仅对向量的指定元素执行。表达式必须采用 `vector[index_vector]` 的形式，因为用任意表达式代替向量名称在这里意义不大。

```bash
> x[is.na(x)] <- 0
```

将 x 中所有缺失值替换为零，

```bash
> y[y < 0] <- -y[y < 0]
```

具有相同的效果

```bash
> y <- abs(y)
```
